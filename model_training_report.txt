================================================================================
  SECTION 1: EXPERIMENTAL DESIGN
================================================================================

  Objective:
  Train and evaluate a Random Forest Classifier to predict the commercial
  success tier of Steam video games using ONLY pre-launch structural features.

  Target Variable: target_tier (estimated ownership brackets)
  Algorithm: Random Forest Classifier
  Split: 80/20 Train/Test (stratified)
  Validation: 5-Fold Stratified Cross Validation

  Two Configurations Tested:
  - Config A: All 66 features including review metrics (pct_pos_total,
    num_reviews_total, peak_ccu)
  - Config B: 62 structural features only (review metrics + release_year removed)

  Rationale for Config B: Review metrics (total reviews, peak concurrent
  players) are generated AFTER a game launches. They are naturally correlated
  with sales volume. Including them risks data leakage, where the model
  uses post-launch proxies instead of learning genuine pre-release patterns.
  Additionally, release_year was excluded as it constitutes temporal metadata
  that cannot be strategically controlled by developers at the pre-launch
  phase, and its inclusion would introduce temporal bias into the model.

  Feature Engineering Evolution:
  The original model used 58 structural features. After a comprehensive
  feature audit, the following changes were made to improve both model
  integrity and predictive performance:

  REMOVED (1 feature):
  - has_reviews: Originally a binary flag derived from num_reviews_total != -1.
    Removed because it is fundamentally derived from post-launch review data,
    creating indirect data leakage. The flag exhibited strong class correlation
    (Flop: 51%, Niche Hit: 82%, Successful: 97%), confirming it acted as a
    proxy for post-launch popularity rather than a genuine structural signal.

  ADDED (5 features):
  - tag_count: Number of Steam user tags assigned to the game. Measures store
    page discoverability and genre breadth. Became the #1 most important
    feature (8.69% importance).
  - screenshot_count: Number of screenshots on the store page. Indicates
    store page quality and developer investment. Ranked #5 (5.71%).
  - has_website: Whether the game has an official website. Signals
    professional presence and marketing investment. Ranked #10 (2.19%).
  - has_workshop: Whether Steam Workshop (modding support) is enabled.
    Developer-controlled community feature decision.
  - required_age: Age rating of the game. Content maturity indicator that
    correlates with game genre and target audience.

  All 5 new features are:
  - 100% available before game launch
  - Developer-controllable decisions
  - Free from data leakage (all correlations with target < 0.3)
  - Independent from existing features (no overlap > 0.7)

  Net result: 58 - 1 + 5 = 62 structural features.

================================================================================
  SECTION 4: DATA LEAKAGE PROOF
================================================================================

  Including post-launch review metrics creates artificial accuracy inflation:

                     With Reviews    Without Reviews    Difference
  Accuracy:            86.06%           77.44%           -8.62%
  Macro-F1:            78.04%           63.48%          -14.56%

  This gap proves that review-based features act as post-launch proxies for
  sales volume, not as genuine pre-launch predictive signals. The PRIMARY
  model deliberately sacrifices this artificial performance to ensure that
  predictions are based entirely on controllable developer decisions.

  Additionally, the has_reviews binary flag was removed from the structural
  feature set because it was derived from num_reviews_total (a post-launch
  metric). Despite being a binary flag rather than a continuous variable,
  it exhibited class-correlated behavior (Flop: 51%, Successful: 97%),
  confirming indirect leakage. Its removal, combined with the addition of
  5 genuinely structural features and hyperparameter tuning (max_depth=25,
  min_samples_split=10, min_samples_leaf=4), resulted in a final Macro-F1
  of 63.48%.
